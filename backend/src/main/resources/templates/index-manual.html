Manual de Usuario - API

Endpoint 1: Get Users

Endpoint: /user/all
Método: GET
Descripción: Llama a todos los usuarios de la base de datos.
Ejemplo: 

                    User{
                        UserID = int;
                        Name = String;
                        Lastnames = String;
                        Email = String;
                        Username = String;
                        Password = String;
                    }

                    int idUsuarioSesion, String token

Endpoint 2: User ID

Endpoint: /user/byid
Método: GET
Descripción: Llama a un usuario existente por ID de la base de datos.
Ejemplo: 

 
                    key: id     
                    int id, String token, int idUsuarioSesion

Endpoint 3: Insertar Usuario

Endpoint: /user
Método: POST
Descripción: Inserta Usuarios a la base de datos.
Ejemplo: 

                    {
                        name: 'Jose',
                        lastnames: 'Madrigal',
                        email: 'jose.m@example.com',
                        username: 'joma',
                        password: 'password123'
                    }

Endpoint 4: Eliminar Usuario

Endpoint: /user
Método: DELETE
Descripción: Borra usuarios de la base de datos.
Ejemplo: 

                    key: id
                    int id, int idUsuarioSesion, String token

Endpoint 5: Crear Nota

Endpoint: /note
Método: GET
Descripción:Crea una nueva nota.
Ejemplo: 

                    
                    String title, String content, int idUsuarioSesion, String token

Endpoint 6: Ver Notas Incompletas

Endpoint: /note/all
Método: GET
Descripción: Devuelve todas las notas incompletas por el ID del usuario.
Ejemplo: 

                    int idUsuarioSesion, String token

Endpoint 7: Actualizar nota por ID

Endpoint: /note/byid
Método: PUT
Descripción: Actualiza una nota existente por su ID
Ejemplo: 

                    
                        id: 1,
                        title: 'Updated Title',
                        content: 'Updated content'
                    
                    int id, int idUsuarioSesion, String token

Endpoint 8: Nota Completada

Endpoint: /note/complete
Método: PUT
Descripción: Cambia el estado de una nota a completada por ID.
Ejemplo: 

                    int id, int idUsuarioSesion, String token

                    id: 3 //Si quieren marcarla como completa deben enviar la id de la nota a la api cuando 
                    marquen el check de completa

Endpoint 9: Ver Notas Completadas

Endpoint: /note/complete
Método: GET
Descripción: Devuelve todas las notas Completadas.
Ejemplo: 

                    key: id value: 3
                    int id, int idUsuarioSesion, String token

Endpoint 10: Delete by ID

Endpoint: /note
Método: DELETE
Descripción: Elimina una nota por ID.
Ejemplo: 

                    key: id, value: 3
                    int id, int idUsuarioSesion, String token

Endpoint 11: Login

Endpoint: /user/login
Método: POST
Descripción: Llama a todos los usuarios de la base de datos.
Ejemplo: 

                    String username, String password
